# 🚀 生產環境部署指南

## 📋 部署前準備

### 1. 資料庫配置
部署環境使用**獨立的資料庫**，與開發環境不共用。

#### 方法一：使用 Neon 資料庫 (推薦)
1. 前往 [Neon.tech](https://neon.tech)
2. 創建新的資料庫專案
3. 複製 `DATABASE_URL` 連接字串

#### 方法二：使用其他 PostgreSQL 提供商
- Supabase
- Railway
- PlanetScale
- 或自架 PostgreSQL

### 2. 環境變數設置

在 Replit Deployments 中設置以下環境變數：

```
DATABASE_URL=postgresql://username:password@host:port/database
```

### 3. 自動初始化功能

程式碼已包含自動初始化功能：
✅ 自動創建多學校 schema
✅ 自動載入示範資料 (新北高中課表)
✅ 自動創建教師和回覆表

### 4. 部署步驟

1. **設置環境變數**：在 Replit Deployments 中配置 `DATABASE_URL`
2. **點擊部署**：系統會自動：
   - 檢測部署環境
   - 初始化資料庫結構
   - 載入示範資料
3. **首次訪問**：系統會完成最終設置

### 5. 驗證部署

訪問部署後的網址：
- `/teacher-portal` - 教師協作回覆系統
- `/multi-school` - 多學校管理後台

## 🔧 常見問題

### Q: 為什麼部署環境出現資料庫錯誤？
A: 部署環境需要獨立配置 `DATABASE_URL`，不會自動使用開發環境的資料庫。

### Q: 資料會從開發環境同步到生產環境嗎？
A: 不會。生產環境會自動載入示範資料，但不會同步開發環境的既有資料。

### Q: 如何確認資料庫正確設置？
A: 訪問 `/api/deployment-test` 查看環境診斷信息。

## 🛡️ 安全注意事項

1. **不要**在程式碼中硬編碼資料庫連接資訊
2. **使用**環境變數存儲敏感資訊
3. **確保**生產資料庫使用強密碼
4. **建議**使用連接池化的 DATABASE_URL (包含 `-pooler`)

## 📞 支援

如遇到問題，請檢查：
1. 環境變數是否正確設置
2. 資料庫連接是否可用
3. 查看部署日誌中的錯誤信息
