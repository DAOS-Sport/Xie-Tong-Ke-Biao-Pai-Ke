# 五泳池課表整合系統｜前後台雙向連通 + 邏輯修補 Prompt
# 原則：不重構、不大幅改動畫面、不新增頁面（只新增區塊/對話框）
# 沿用現有 shadcn/ui 元件、Tailwind CSS、Drizzle ORM 架構

---

## 核心方針

本次修改聚焦三件事：
1. **補指派通知**：教練被排課後一定要知道
2. **補填寫可見性**：管理員要看到哪些教練還沒填時段/偏好
3. **補邏輯連貫**：Ragic 同步、鎖定流程、自動填入的邏輯斷點

---

## SWIM-01：教練指派後 LINE 推播通知

**斷點：** 管理員在 Phase 2（教練指派）指派或更換教練後，教練完全不知道。

**後端：**
在以下三個操作觸發後，自動發送 LINE push：
1. `PATCH /api/schedules/:id`（更新 coachName 或 coachName2）
2. `POST /api/admin/auto-fill`（自動填入後）
3. 解鎖後重新指派（isClassLocked 從 true → false 再變更 coach）

```ts
// 在 schedule 更新時，找出受影響的教練並推播
async function notifyCoachAssigned(
  schedule: Schedule,
  previousCoach1: string | null,
  previousCoach2: string | null
) {
  const affectedCoaches = []

  // 新指派的教練
  if (schedule.coachName && schedule.coachName !== previousCoach1) {
    affectedCoaches.push({
      name: schedule.coachName,
      role: schedule.coach1IsTeaching ? '當班教學' : '教練'
    })
  }
  if (schedule.coachName2 && schedule.coachName2 !== previousCoach2) {
    affectedCoaches.push({
      name: schedule.coachName2,
      role: schedule.coach2IsTeaching ? '當班教學' : '協助'
    })
  }

  for (const { name, role } of affectedCoaches) {
    const coachUser = await db.query.coachUsers.findFirst({
      where: and(
        eq(coachUsers.linkedCoachName, name),
        isNotNull(coachUsers.lineId)
      )
    })
    if (!coachUser?.lineId) continue

    const dateStr = format(parseISO(schedule.date), 'M/d (EEE)', { locale: zhTW })
    const timeSlotLabel = await getTimeSlotLabel(schedule.timeSlotId)

    await sendLinePush(coachUser.lineId,
      `📋 課程指派通知\n` +
      `您已被排入以下課程：\n` +
      `日期：${dateStr}\n` +
      `時段：${timeSlotLabel}\n` +
      `場館：${schedule.venueName}\n` +
      `班級：${schedule.className}\n` +
      `角色：${role}\n\n` +
      `請前往教練端確認：${process.env.COACH_PORTAL_URL}`
    )
  }
}
```

**自動填入（auto-fill）的批次通知：**
```ts
// auto-fill 完成後，彙整所有新指派並統一發送
// 避免同一教練收到多則獨立通知，改為一則彙整訊息
const coachSummary = new Map<string, string[]>() // lineId → 課程描述[]

for (const assignment of newAssignments) {
  // 彙整同一教練的所有新課
  if (!coachSummary.has(assignment.lineId)) {
    coachSummary.set(assignment.lineId, [])
  }
  coachSummary.get(assignment.lineId)!.push(
    `${assignment.dateStr} ${assignment.timeSlot} @${assignment.venue}`
  )
}

// 每位教練只發一則彙整訊息
for (const [lineId, courses] of coachSummary) {
  await sendLinePush(lineId,
    `📋 課程指派通知（本週新增 ${courses.length} 堂）\n` +
    courses.map((c, i) => `${i+1}. ${c}`).join('\n') +
    `\n\n請前往教練端確認課表。`
  )
}
```

**驗收：**
1. 手動指派教練後，該教練收到 LINE 通知
2. 自動填入後每位受影響教練收到彙整通知
3. 被移除的教練（從某堂課中移除）也收到通知

## SWIM-02：課表異動通知（鎖定/修改後）

**斷點：** Phase 1 鎖定或解鎖後，已指派的教練不會收到任何通知。
班名或時段異動後教練繼續以舊資料為準。

**後端：**
在以下操作後觸發通知：
1. `POST /api/schedules/lock`（鎖定 venue+week）
2. `POST /api/schedules/unlock`（解鎖）
3. Phase 1 修改 className 後（若已有教練指派）

```ts
// 解鎖時，通知該 venue+week 所有已指派的教練
async function notifyScheduleUnlocked(venueName: string, weekStart: string) {
  const affectedSchedules = await db.query.schedules.findMany({
    where: and(
      eq(schedules.venueName, venueName),
      between(schedules.date, weekStart, addDays(weekStart, 6))
    )
  })

  const coachLineIds = new Set<string>()
  for (const s of affectedSchedules) {
    // 收集所有有 lineId 的教練
    for (const name of [s.coachName, s.coachName2].filter(Boolean)) {
      const c = await findCoachLineId(name!)
      if (c) coachLineIds.add(c)
    }
  }

  for (const lineId of coachLineIds) {
    await sendLinePush(lineId,
      `⚠️ 課表異動通知\n` +
      `${venueName} ${format(parseISO(weekStart),'M/d')} 週的課表已更新，` +
      `請重新確認您的排課內容。\n\n` +
      `建議重新匯出 Google Calendar。`
    )
  }
}
```

## SWIM-05：自動填入邏輯強化（場館偏好優先）

**斷點：** 目前自動填入優先考量「排課數最少」（負載均衡），
但場館偏好的權重未明確設定，可能派教練去他不能去的場館。

**後端（`/api/admin/auto-fill` 或 auto-fill 邏輯）：**
```ts
// 修改候選教練的篩選順序
async function getCandidateCoaches(
  venueName: string,
  date: string,
  timeSlotId: number
): Promise<RankedCoach[]> {

  const dayOfWeek = getDay(parseISO(date)) || 7  // 1=Mon, 7=Sun
  const timeSlotOrder = await getTimeSlotOrder(timeSlotId)

  // Step 1: 找有填「該時段可用」的教練
  const available = await db.query.coachAvailability.findMany({
    where: and(
      eq(coachAvailability.dayOfWeek, dayOfWeek),
      eq(coachAvailability.timeSlotOrder, timeSlotOrder)
    )
  })
  const availableNames = new Set(available.map(a => a.coachName))

  // Step 2: 找有「該場館偏好」的教練
  const prefersVenue = await db.query.coachVenuePreferences.findMany({
    where: eq(coachVenuePreferences.venueName, venueName)
  })
  const prefersNames = new Set(prefersVenue.map(p => p.coachName))

  // Step 3: 確認無衝突（同時段未排其他課）
  const allApproved = await db.query.coachUsers.findMany({
    where: eq(coachUsers.status, 'approved')
  })

  const ranked = allApproved.map(coach => {
    const name = coach.linkedCoachName ?? coach.name
    const isAvailable = availableNames.has(name)
    const prefersThisVenue = prefersNames.has(name)

    // 優先級分數
    let score = 0
    if (isAvailable && prefersThisVenue) score = 3   // 最優先
    else if (isAvailable) score = 2                   // 有時段但無場館偏好
    else if (prefersThisVenue) score = 1              // 有場館偏好但時段未填
    else score = 0                                     // 都沒填

    return { name, score, isAvailable, prefersThisVenue }
  })

  // score=0 的教練不自動填入（兩樣都沒填的不強派）
  return ranked
    .filter(c => c.score > 0)
    .sort((a, b) => b.score - a.score || getAssignedCount(a.name) - getAssignedCount(b.name))
}
```

在管理員的指派 UI 中，候選教練顯示標記更新：
- `✅🏊` = 有時段 + 有場館偏好（最優）
- `✅` = 只有時段
- `🏊` = 只有場館偏好
- 無標記 = 兩樣都沒填（仍可手動指派，但自動填入不考慮）

**驗收：**
1. 自動填入優先選「有時段 + 有場館偏好」的教練
2. 兩樣都沒填的教練不會被自動指派
3. 手動指派界面的標記更新為三種類型

## SWIM-06：Ragic 同步保護機制

**斷點：** Ragic 每日 03:00 自動同步，可能覆蓋管理員手動修改的教練資料
（目前設計為「不刪除不覆寫」，但需確認 lineId 等欄位有保護）。

**後端（`server/ragic.ts`）：**
確認同步邏輯對以下欄位有保護：
```ts
// 同步教練資料時，lineId 欄位只在「目前為空」時才更新
// （已在架構中提及 isValidLineUserId 保護，確認實際執行）
if (existingCoach) {
  await db.update(coachUsers)
    .set({
      name: ragicData.name,
      phone: ragicData.phone,
      email: ragicData.email,
      // lineId：只在目前為空或無效時才更新
      ...(isValidLineId(existingCoach.lineId)
        ? {}  // 已有有效 lineId，不覆蓋
        : { lineId: ragicData.lineId ?? null }
      ),
      // employeeId：只在目前為空時填入（架構中已提及）
      ...(existingCoach.employeeId
        ? {}
        : { employeeId: ragicData.employeeId ?? null }
      ),
    })
    .where(eq(coachUsers.id, existingCoach.id))
}
```

新增同步日誌欄位（可選，建議）：
```sql
ALTER TABLE coachUsers ADD COLUMN last_ragic_sync timestamptz;
ALTER TABLE coachUsers ADD COLUMN ragic_sync_note text;
```

管理後台的「Ragic 同步狀態」顯示中，列出最後一次同步有哪些教練資料被更新。

**驗收：**
1. 已有有效 lineId 的教練，Ragic 同步後 lineId 不被覆蓋
2. 管理後台可查看最後一次 Ragic 同步的更新摘要

在統計頁面頂部新增「月份彙整」快速模式（toggle）：
```tsx
<div className="flex items-center gap-3 mb-4">
  <span className="text-sm text-gray-500">檢視模式</span>
  <div className="flex rounded-lg border border-gray-200 overflow-hidden">
    <button
      className={cn("px-3 py-1 text-sm", viewMode==='week' && "bg-gray-100 font-medium")}
      onClick={() => setViewMode('week')}
    >按週</button>
    <button
      className={cn("px-3 py-1 text-sm", viewMode==='month' && "bg-gray-100 font-medium")}
      onClick={() => setViewMode('month')}
    >按月</button>
  </div>
</div>
```

月份模式下，後端 `GET /api/stats?month=yyyy-MM` 自動彙整四週數據，
Excel 匯出也支援月份模式。
